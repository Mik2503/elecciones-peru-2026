6. PROBLEMA DE BLOQUEO ONPE DESDE VERCEL (ERROR 403)
================================================================================

DIAGNÓSTICO:
────────────
El firewall de la ONPE (probablemente Cloudflare o similar) bloquea
activamente las peticiones que provienen de IPs de servidores cloud como
Vercel. Esto se evidencia con un error HTTP 403 (Forbidden).

Los headers personalizados (User-Agent, Referer, Origin) implementados en
/api/fetch-onpe NO son suficientes para evadir el bloqueo.

POR QUÉ SUCEDE:
───────────────
  1. Las IPs de Vercel son públicas y conocidas por los firewalls
  2. La ONPE puede tener políticas anti-scraping que bloquean rangos cloud
  3. El User-Agent de Chrome no engaña a firewalls de nivel empresarial
  4. El Referer/Origin se pueden verificar pero no son autenticación real

SOLUCIONES POSIBLES (ordenadas por viabilidad):

  OPCIÓN A - Proxy Residencial (Recomendada):
    Usar un servicio proxy con IPs residenciales (BrightData, SmartProxy,
    ScraperAPI, etc.) que rote las peticiones a través de IPs de Perú.
    → Costo: ~$50-100/mes
    → Ventaja: Funciona garantizado
    → Desventaja: Costo recurrente

  OPCIÓN B - Servidor Intermedio en Perú:
    Desplegar un pequeño servidor (VPS barato) con IP residencial peruana
    que haga de proxy entre Vercel y la ONPE.
    → Costo: ~$5-10/mes
    → Ventaja: Control total, bajo costo
    → Desventaja: Infraestructura adicional

  OPCIÓN C - Web Scraping con Puppeteer/Playwright:
    Usar un navegador headless real en lugar de fetch() simple.
    → Costo: Gratis (pero consume más recursos)
    → Ventaja: Simula un navegador real
    → Desventaja: Puede que Cloudflare también lo detecte

  OPCIÓN D - API Oficial con Credenciales:
    Contactar a la ONPE para obtener acceso autorizado a su API.
    → Costo: Depende de la ONPE
    → Ventaja: Solución oficial y estable
    → Desventaja: Burocracia, puede no estar disponible

  OPCIÓN E - Usar el Docker Sync desde IP residencial:
    Ejecutar el contenedor Docker desde una máquina con IP de Perú
    (servidor local en Perú, Raspberry Pi, etc.)
    → Costo: Variable
    → Ventaja: Ya está implementado el script
    → Desventaja: Necesitas infraestructura local

8. PROBLEMAS DETECTADOS Y RECOMENDACIONES
================================================================================

PROBLEMA CRÍTICO #1: Bloqueo de firewall ONPE desde Vercel
─────────────────────────────────────────────────────────────
Severidad: ALTA
Impacto: No se pueden obtener datos desde los servidores de Vercel
Solución: Ver Opciones A-E en sección 6

PROBLEMA CRÍTICO #2: Inconsistencia en sync_official.sh
─────────────────────────────────────────────────────────────
Severidad: ALTA
El script apunta API_ENDPOINT a /api/fetch-onpe, pero ese endpoint
SOLO tiene método GET, no POST. El script hace curl -X POST.
Solución: Cambiar API_ENDPOINT a /api/fetch-results (que SÍ acepta POST)

PROBLEMA MEDIO #3: Cron job insuficiente
─────────────────────────────────────────────────────────────
Severidad: MEDIA
El cron de Vercel corre solo 1 vez al día. Para elecciones en tiempo
real, se necesita al menos cada 1-5 minutos.
Solución: Usar Docker Sync o actualizar vercel.json con cron más frecuente
  (nota: los crons de Vercel tienen limitaciones en plan gratuito)

PROBLEMA MENOR #4: No hay fetch automático al iniciar
─────────────────────────────────────────────────────────────
Severidad: BAJA
Si KV está vacío, /api/data retorna null sin intentar fetch automático.
Solución: Agregar lógica en /api/data para trigger automático

PROBLEMA MENOR #5: Colores hardcodeados limitados
─────────────────────────────────────────────────────────────
Severidad: BAJA
Solo 6 partidos están mapeados en getPartyColor(). Si hay más partidos
en la boleta, todos tendrán el color gris por defecto.
Solución: Generar colores dinámicos o expandir el mapa

PROBLEMA MENOR #6: Sin SSR
─────────────────────────────────────────────────────────────
Severidad: BAJA
El dashboard es 100% client-side, lo que afecta SEO y tiempo de carga
inicial.
Solución: Migrar a Server Components con fetch en servidor
